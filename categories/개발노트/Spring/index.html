Embeddable & @Embedded

Mar 20, 2021 in 개발노트, Spring

@Embeddable @Embedded

하나로 표현하고자 하는 개념에 해당하는 클래스를 생성한 뒤에 필드를 생성하고 그 후
클래스에 @Embeddable을 달아줍니다! (이 과정은 생략해도 무방합니다)

마지막으로 Entity 클래스에서 앞서 생성한 객체 필드를 생성하고, 필드 바로위에
@Embedded을 선언해 줍니다!

@AttributeOverride

@Embedded를 이용해 객체로 엔티티의 컬럼을 표현하면면 컬럼 이름이 중복되는 상황이 일어납니다.
이럴 경우에 @AttributeOverride 어노테이션을 사용합니다.

@AttributeOverride (name = “기존 컬럼”, column = @column ( name = “ 특정 컬럼 이름 “ ) )
으로 선언하여 사용이 가능합니다!

Embedded 타입 사용시

FrontEnd에서 이와같은 임베디드 타입 사용시 임베디드 타입 자체를 메모리에 올려줘야 사용이 가능하다

ex)
this.state.fuseRquest.manualImage = {}

백엔드에서 임베디드 타입에 접근 할 때, 이미 인베디드 타입 자체가 NULL이면 그 내영자체가 들어가기 전에 NPE에러를 발생하기 때문에 임베디드 타입 자체를 따로 null Check를 해줘야 한다.
