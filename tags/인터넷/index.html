HTTP & HTTPS

Mar 20, 2021 in 이론노트, 인터넷

HTTP vs HTTPS

-> html을 전송하기 위한 규약
-> S는 Security의 약자

HTTP vs SSL

인터넷의 계층 구조는 TCP/IP -> SSL/TLS -> HTTP 순이다
즉 HTTP가 SSL을 쓰는게 HTTPS이다!

SSL vs TLS

SSL이 상용화 되면서 이것을 인터넷 기준으로 표준화 한것을 TLS라고 한다 , 즉 같은것이다!

SSL 인증서

클라이언트가 서버에 접속하면 서버가 클라이언트에게 SSL 인증서를 보여준다.
클라이언트는 이 인증서를 검증/절차 과정을 거치며 이 과정을 거치면 해당 사이트를 진행 시킨다

역할

  1. 공격자에게 노출되는 것을 막는다

1.. 대칭키 보완

암호화 복호화를 진행 할 때 하나의 인증키를 사용한다. 보통 개인의 것을 보관 할 떄 많이 사용하며 해당 파일을 다른 파일로 보낼 때 대칭키를 보내야 함으로 해킹의 위험도가 높다

2.. 공개키 보완하나의 파일을 암호화 할 때 2개의 쌍 키를 만든다. 공개키로 암호화를 진행하고 개인키로 복호화를 진행한다.

  1. 신뢰성 있는 사이트인지 판단한다.
  2. 악의적인 변경을 하지 못하게 한다.
  3. SSL 통신에 필요한 공개키를 클라이언트에게 제공한다.

이러한 SSL 인증서는 CA라고하는 기업이 책임지고 인증서를 발급 해준다. 이같은 인증서는 신뢰가 있는 사이트인지 판별 할 수 있도록 도와준다

내가 개인적으로 사이트를 운영 하고 있다면 이러한 CA기관에 자료를 주고 내가 안전한 사람이라고 판별되면 인증서가 나오는데 그것을 구매하고 사용하면 된다!

SSL 인증서에 포함되어 있는것

  1. 서비스의 정보 ( CA의 정보, 어떤 서비스를 하는지 )
  2. 서버의 공개키 ( 공개키 ,암호화 방법 )

브라우저는 모든 CA의 정보를 알고 있기 때문에 이 같은 SSL 인증서를 받으면 CA의 리스트에 있는지 판단하고 맞으면 해당 개인키로 복호화를 진행하고 이를 성공하면 신뢰성 있는 사이트라고 판단한다.

복원하는 방법

접속 -> 서버가 인증서 전달 -> 브라우저가 CA확인 -> CA의 개인키로 복호화 -> 해당 CA인거를 인증서로 인증

hosting & DNS

Mar 20, 2021 in 이론노트, 인터넷

DNS

host

host란 네트워크를 가질 때 가질 수 있는 것

host란 네트워 크를 가질 때 가지는 것이다.
그러면 각 host는 이름을 가지게 되고 그것이 바로 ip주소이다.
이런 ip주소를 알고 있으면 다른 컴퓨터에서 접속이 가능 할 수 있게된다!

이러한 ip주소들은 이름이 어렵기 때문에 사람이 기억하기 쉽게 바꾸기를 원한다. 이러한 별명(?)을 저정하는 곳이 바로 DNS Server이고 이러한 별명짓기를 DNS라고 부른다!

DNS Server를 사용하지 않을 때에는 자신의 컴퓨터에 저장되어 있는 hosts file에 DNS를 저장 할 수 있다.

DNS Server

그래서 S.R.I가 전체가 사용하는 hosts를 관리했었다. 그래서 호스트를 등록하고 싶은 사람은 SRI에게 전화를 해서 등록을 하였고,
모든 사람들은 SRI에서 hosts를 다운 받아서 사용했었다.
이러한 방법은 시간과 비용이 많이 들었을 뿐더러 파일로 저장하기 때문에 용량의 한계가 있었다.

이러한 한계를 극복하기 위해 나온것이 DNS Server이다.
우리는 내 컴퓨터안의 hosts에 저장하는것이 아닌 DNS Server에 저장함으로써 기존의 방법을 자동화 하였고 파일에 저장하던 것을
서버로 저장함으로써 용령의 한계를 극복하였다.

이러한 DNS Server는 하나만 있는것이 아니라 여러개가 존재한다. 이러한 것을 public DNS Server라고 한다.

DNS의 내부구조

DNS는 root -> Top-level -> second-level -> sub 순으로 이어진 다
각 레벨별로 담당하는 server가 존재하고
직속 하위 server에 대한 정보는 필수적으로 가지고 있다
우리가 dns을 호출 할 때는 root부터 밑에까지 차례대로 호출하면서 알려준다.

DNS registor

ICANN - Root ( a.Root.name.server ) [ com NS a.grid.servers.net ]
Registry - Top-level-domain ( a.grid.server.net ) [ 등록자에 의해 붙음 ]

등록자 -> server 개설 -> 등록대행자 (Registrar) -> Registry [ example.com A 93.454.345 ]

접속자 -> hosts -> Root server -> Registry -> 새로운 등록된 사이트 접속

DNS recode

A -> Ip주소 앞에 나오는 명령어
Ns -> 특정 도메인 이름을 저장
CNAME -> 도메인에 대한 다른 도메인을 가르킴 !

—> 각각 하나가 DNS recode라고 부른다

Example.com -> A -> 192.123.12.23
www.example.com -> CNAME -> example.com

-> A의 아이피 주소가 주기적으로 바뀔때 사용한다!

도메인 vs URL

도메인은 DNS Server가 관리하고 있는 서버이고,
URL은 그 서버 안에 접근 할 때 쓰는 주소이다!

DNS Forwarding

  1. 유동 포워딩 : 다른 사이트로 옮기면 주소가 바뀐다
  2. 고정 포워딩 : 다른 사이트로 옮겨도 주소가 바뀌지 않는다

브라우저의 주요 기능

Mar 20, 2021 in 이론노트, 인터넷

브라우저의 주요 기능

브라우저의 주요 기능은 사용자가 선택한 자원을 서버에 요청하고 브라우저에 표시하는 것이다.
자원의 주소는 URI에 의해 정해진다. 브라우저는 웹 표준화 기구인 W3C(World Wide Web Consortium)에서 정한 명세에 따라 파일을 해석해서 표시한다.

브라우저의 구성요소

  1. 유저 인터페이스 : 이전, 다음, 새로고침, 북마크 같이 요청한 페이지는 보여주는 부분 외에 유저에게 동일하게 보이는 부분이다.

  2. 브라우저 엔진 : 유저 인터페이스와 렌더링 엔진 사이에 쿼리를 전달할 수 있게 조작을 담당합니다.

  3. 렌더링 엔진 : 사용가자가 요청한 페이지는 화면에 표시하기 위해 HTML과 CSS를 파싱/해석합니다.

(속도와 관련해서 가장 궁금했던 부분!! 크롬의 경우 각 탭마다 별도의 렌더링 엔진 인스턴스를 유지하여 독립된 프로세스로 처리된다고 합니다.)

  1. 네트워킹 : http 요청을 할 수 있고, 네트워크 호출에 사용됩니다.

  2. 자바스크립트 해석기 : Javascript코드를 해석 및 실행합니다.

  3. UI 벡엔드 : 기본적인 UI들 예로 input, select 같은 위젯을 그리는 인터페이스입니다.

  4. 자료 저장소 : 쿠키같은 지원들을 저장하는 영역이다.

렌더링 엔진

사용자가 요청한 내용을 서버로부터 받아 브라우저 화면에 표시하는 역할을 함
(1) 표시할 수 있는 것 : HTML 및 XML 문서 그리고 이미지를 표시할 수 있음
(2) 확장 : 플러그인이나 브라우저마다 확장 기능을 이용해 PDF와 같은 유형도 표시할 수 있음
(3) 종류 : 사파리, 크롬은 웹킷(Webkit)엔진(크롬은 28버젼부터 Blink) / 파이어폭스는 모질라에서 만든 게코(Gecko)엔진
(4) 동작원리
기본적인 동작 과정
HTML 문서를 파싱 > 브라우저 화면에 랜더링하기위해 다루기 쉬운 구조로 바꿈 > css파일 파싱 > 렌더트리 구축 > 렌더트리 배치 > 렌더트리 그리기

스타일 정보와 HTML 표시 규칙은 렌더 트리라고 부르는 또 다른 트리를 생성함.
화면에 표시하기위한 위치, 크기, 그리기 순서 저장을 위한 별도의 트리 구조(렌더트리)가 필요함.
렌더링 엔진은 좀 더 나은 사용자 경험을 위해 가능하면 빠르게 내용을 표시하는데 모든 HTML을 파싱할 때까지 기다리지 않고 배치와 그리기 과정을 시작한다. 네트워크로부터 나머지 내용이 전송되기를 기다리는 동시에 받은 내용의 일부를 먼저 화면에 표시하는 것이다.

인터넷은 어떻게 동작하는가?

Mar 20, 2021 in 이론노트, 인터넷

인터넷은 어떻게 동작하는가?

  1. 인터넷 초기에는 컴퓨터와 컴퓨터 1:1의 연결을 위해서 유선연결을 하여서 서로의 데이터를 주고 받았다.

  2. 시대가 발전하면서 1:1의 연결이 아닌 N:N의 연결이 필요하게 되면서 많은 연결 케이블이 필요하게 되었는데. 이를 방지하기 위해 라우터라는 하나의 컴퓨터를 만들어서 ( 이것은 컴퓨터간의 전송 및 수신을 확인하는 용도로만 사용함) 케이블의 수를 줄였다.

  3. 하나의 라우터로는 한계가 있기 때문에 여러개의 라우터를 이용하였다.

  4. 라우터로 하게 된다면 거리가 아주 먼 거리에 있는 곳 까지 연결하기 위해서는 거기까지 케이블을 깔아야 했다 이를 방지하지 위해서 집집마다 깔려져있는 전화국을 이용하기로 했는데 인터넷 신호를 전화국 신호로 옮기기위한 장치가 모뎀이다.

  5. 모뎀을 통해 우리의 네트워크는 전화 시설에 연결된다. 하지만, 아직까지 우리의 컴퓨터가 보낸 메시지가 도달해야 할 컴퓨터(혹은 네트워크)까지 도달하지 않은 상태다. 이 메시지가 전달되려면 인터넷 서비스 제공 업체(ISP, Internet Service Provider)에 연결되어야 한다.
